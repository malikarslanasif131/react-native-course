What is Redux Toolkit and how does it simplify Redux?

📖 Answer

Redux Toolkit (often called RTK) is the officially recommended way to write Redux today — it wraps the classic Redux patterns (actions, reducers, store) in a much smaller, less repetitive API. Instead of hand-writing action type constants, action creators, and switch-statement reducers separately, RTK lets you define all three together in one place, and it includes sensible defaults (like Immer for "mutating" state safely) baked in. It's the difference between assembling furniture from 40 loose parts with a separate instruction booklet (classic Redux) versus a kit where most of the joints are pre-fitted and only need a few screws (RTK).

Bash · Install 
npm install @reduxjs/toolkit react-redux
// Replaces hand-written actions + reducers + store setup with one API
182

What is createSlice in Redux Toolkit?

📖 Answer

createSlice bundles together a piece of state's initial value, its reducer logic, and the action creators that trigger that logic — all generated automatically from one object you define. You write what looks like simple, "mutating" code inside each reducer function (e.g., state.count += 1), and RTK safely converts it into proper immutable updates behind the scenes using Immer. This collapses what used to be three separate files (action types, action creators, reducer) into a single, cohesive definition.

JS · createSlice 
import { createSlice } from '@reduxjs/toolkit';

const counterSlice = createSlice({
  name: 'counter',
  initialState: { value: 0 },
  reducers: {
    increment: (state) => { state.value += 1; }, // looks mutating, isn't
  },
});

export const { increment } = counterSlice.actions;
export default counterSlice.reducer;

What is RTK Query, and how does it relate to TanStack Query?

📖 Answer

RTK Query is Redux Toolkit's built-in solution for fetching and caching server data — solving the exact same "server state" problem TanStack Query does (caching, refetching, loading/error states), but integrated directly into the Redux store so the cached data lives alongside your other Redux state. If a team is already committed to Redux for client state, RTK Query avoids introducing a second, separate data-fetching library; if a team has no Redux dependency at all, TanStack Query alone is usually simpler to adopt. They solve the same problem from two different architectural starting points.

JS · RTK Query Endpoint 
import { createApi, fetchBaseQuery } from '@reduxjs/toolkit/query/react';

export const api = createApi({
  baseQuery: fetchBaseQuery({ baseUrl: 'https://api.example.com' }),
  endpoints: (builder) => ({
    getUsers: builder.query({ query: () => 'users' }),
  }),
});

export const { useGetUsersQuery } = api; // generated hook, just like useQuery

What is react-hook-form and why is it popular for forms in React Native?

📖 Answer

react-hook-form manages form state and validation while minimizing re-renders — instead of a re-render on every single keystroke across every field (which manual useState per field would cause), it tracks input values internally via refs and only re-renders when truly necessary, like on submit or validation error. You register each field with its control/register API, and it handles gathering values, running validation, and reporting errors for you. For forms with many fields, this can meaningfully improve performance compared to wiring up individual state for every input by hand.

JSX · Basic react-hook-form 
import { useForm, Controller } from 'react-hook-form';

const { control, handleSubmit } = useForm();

<Controller
  control={control}
  name="email"
  render={({ field }) => <TextInput value={field.value} onChangeText={field.onChange} />}
/>
<Button onPress={handleSubmit((data) => console.log(data))} title="Submit" />
185

What is form validation with Zod, and how does it pair with react-hook-form?

📖 Answer

Zod is a schema-validation library — you describe the exact "shape" your data should have (a string, an email format, a number with a minimum) once, and Zod can validate any object against it, producing clear error messages when something doesn't match. Paired with react-hook-form via a resolver, your validation rules live in one declarative schema instead of scattered conditional checks throughout your form's JSX — the form library handles re-render efficiency, and Zod handles "is this data actually valid," each doing the job it's best at.

JS · Zod + react-hook-form 
import { z } from 'zod';
import { zodResolver } from '@hookform/resolvers/zod';

const schema = z.object({
  email: z.string().email('Invalid email address'),
});

const { control } = useForm({ resolver: zodResolver(schema) });

What is i18n (internationalization), and how do you implement it in React Native?

📖 Answer

i18n is the practice of structuring your app so its text can be displayed in multiple languages without rewriting components — every piece of user-facing text is replaced with a "key" (like welcome.title), and a translation file maps that key to the actual phrase in each supported language. A library like i18next (with react-i18next) handles loading the right translation file and gives you a t() function to look up the current phrase by key, anywhere in your component tree.

JSX · Using a Translation Key 
import { useTranslation } from 'react-i18next';

function Welcome() {
  const { t } = useTranslation();
  return <Text>{t('welcome.title')}</Text>;
  // en.json: { "welcome": { "title": "Welcome!" } }
  // fr.json: { "welcome": { "title": "Bienvenue !" } }
}

How does expo-localization help detect a user's locale and preferred language?

📖 Answer

expo-localization reads the device's own system settings — the language and region the user has already configured on their phone — so your app can automatically pick a sensible default language on first launch, rather than forcing the user to manually select one. It returns details like the device's locale code (en-US, fr-FR) and calendar/region preferences, which you typically feed straight into your i18n library's initial language setting.

JS · Reading Device Locale 
import * as Localization from 'expo-localization';

console.log(Localization.locale);       // e.g. "en-US"
console.log(Localization.locales);      // user's full preference list

i18n.changeLanguage(Localization.locale.split('-')[0]); // "en"

What is expo-camera and how do you take a photo with it?

📖 Answer

expo-camera renders a live camera preview directly inside your app's UI and gives you a method to capture a still photo (or record video) from that feed, after requesting camera permission. You attach a ref to the <CameraView> component, and calling takePictureAsync() on it returns the captured photo's local file URI, which you can then display, upload, or save. It's the building block behind any in-app photo capture feature — profile pictures, document scanning, social posts.

JSX · Capturing a Photo 
import { CameraView } from 'expo-camera';

const cameraRef = useRef(null);

async function takePhoto() {
  const photo = await cameraRef.current.takePictureAsync();
  console.log(photo.uri); // local file path to the captured image
}

<CameraView ref={cameraRef} style={{ flex: 1 }} />

What is expo-image-picker and how do you let a user choose a photo from their library?

📖 Answer

expo-image-picker opens the device's native photo/video library picker (or the camera directly) and resolves with the file the user selected — no custom UI needed, since it's the same familiar gallery interface the user already knows from their phone's Photos app. This requires media library permission first, and the result includes the selected asset's local URI, dimensions, and file type, ready to upload or display.

JS · Picking an Image 
import * as ImagePicker from 'expo-image-picker';

const result = await ImagePicker.launchImageLibraryAsync({
  mediaTypes: ImagePicker.MediaTypeOptions.Images,
  quality: 0.8,
});

if (!result.canceled) {
  console.log(result.assets[0].uri);
}

How do you scan a QR code or barcode in React Native?

📖 Answer

Barcode/QR scanning is built directly into the same expo-camera module used for photos — you pass an onBarcodeScanned callback and a list of barcodeTypes to scan for, and the camera view automatically fires that callback the instant it recognizes a matching pattern in the live feed, returning the decoded string data. There's no separate "scanning engine" you need to manage — the camera component is doing real-time image analysis on every frame internally.

JSX · QR Scanning 
<CameraView
  style={{ flex: 1 }}
  barcodeScannerSettings={{ barcodeTypes: ['qr'] }}
  onBarcodeScanned={({ data }) => {

  }}
/>

What is expo-local-authentication and how do you trigger Face ID/Touch ID?

📖 Answer

expo-local-authentication taps into the device's built-in biometric hardware — Face ID or Touch ID on iOS, fingerprint or face unlock on Android — letting your app prompt the user to authenticate without ever seeing or handling the actual biometric data itself (that stays sealed inside the device's secure hardware). You first check hasHardwareAsync() and isEnrolledAsync() to confirm the device supports it and the user has set it up, then call authenticateAsync() to show the native prompt and get a pass/fail result.

JS · Triggering Biometric Auth 
import * as LocalAuthentication from 'expo-local-authentication';

const result = await LocalAuthentication.authenticateAsync({
  promptMessage: 'Unlock with Face ID',
});

if (result.success) {
  // Proceed — user verified biometrically
}

What is expo-haptics, and when should you use haptic feedback?

📖 Answer

expo-haptics triggers the device's small physical vibration motor in different patterns — a light tap, a medium thud, a success/warning/error "buzz" — to give the user a physical confirmation that something happened, supplementing (not replacing) visual feedback. Good use cases: confirming a successful form submission, signaling an error, or giving tactile feedback on a custom slider/switch. Overusing it on every minor interaction quickly feels gimmicky or annoying, so it's best reserved for genuinely meaningful moments.

JS · Triggering Haptic Feedback 
import * as Haptics from 'expo-haptics';

await Haptics.notificationAsync(Haptics.NotificationFeedbackType.Success);
await Haptics.impactAsync(Haptics.ImpactFeedbackStyle.Light);

What is expo-clipboard, and how do you copy/paste text?

📖 Answer

expo-clipboard reads from and writes to the device's system clipboard — the same shared clipboard the user's keyboard "paste" action uses. setStringAsync() copies a string onto the clipboard (e.g., "Copy referral code" buttons), and getStringAsync() reads back whatever's currently there. It's a small but commonly requested feature, especially for sharing codes, links, or copied confirmation numbers.

JS · Copy & Read Clipboard 
import * as Clipboard from 'expo-clipboard';

await Clipboard.setStringAsync('REF-12345');     // copy
const text = await Clipboard.getStringAsync();   // read back

What is expo-file-system used for?

📖 Answer

expo-file-system lets you read, write, move, copy, and delete files directly on the device's local storage — useful for caching downloaded data, saving a generated PDF, or storing larger files that don't belong in a simple key-value store like AsyncStorage. It exposes paths to safe, sandboxed app-specific directories (a "documents" directory and a "cache" directory) so you're not poking around the rest of the device's file system, which the OS wouldn't allow anyway for security reasons.

JS · Writing & Reading a File 
import * as FileSystem from 'expo-file-system';

const path = FileSystem.documentDirectory + 'notes.txt';
await FileSystem.writeAsStringAsync(path, 'Hello, file system!');
const contents = await FileSystem.readAsStringAsync(path);

What is expo-sharing and how do you let users share content from your app?

📖 Answer

expo-sharing opens the device's native share sheet — the same panel that pops up when sharing a photo or link from any app, listing Messages, Mail, social apps, AirDrop, etc. You give it a local file URI (an image, PDF, or other document already saved on-device), and the OS handles presenting all the apps capable of receiving that file type, with zero custom UI needed on your end. It's a thin wrapper that hands the entire "who do you want to share this with" decision over to the platform's own polished, familiar interface.

JS · Native Share Sheet 
import * as Sharing from 'expo-sharing';

if (await Sharing.isAvailableAsync()) {
  await Sharing.shareAsync(fileUri); // opens native share sheet
}

What is expo-av (or expo-audio/expo-video) used for?

📖 Answer

These modules provide native audio and video playback and recording — playing a sound effect, streaming a video file, or recording audio from the microphone — wrapping the platform's underlying media engines so you don't write any native code yourself. What is @gorhom/bottom-sheet, and why is it preferred over a custom Modal for this pattern?

📖 Answer

A bottom sheet is a panel that slides up from the bottom of the screen and can be dragged up/down or snapped to different height "stops" — common in apps like Maps or Uber for showing details without leaving the current screen. @gorhom/bottom-sheet handles the tricky gesture physics (drag velocity, snap points, backdrop dimming, keyboard avoidance) that would be genuinely difficult to build correctly from scratch with a plain Modal, which has no built-in concept of partial height or drag-to-resize at all.

JSX · Basic Bottom Sheet 
import BottomSheet from '@gorhom/bottom-sheet';

const sheetRef = useRef(null);

<BottomSheet ref={sheetRef} snapPoints={['25%', '50%', '90%']}>
  <Text>Drag me up and down!</Text>
</BottomSheet>

What is a Toast/Snackbar, and how do you implement one in React Native?

📖 Answer

A Toast (or Snackbar) is a small, temporary message that appears briefly — usually near the top or bottom of the screen — to confirm an action ("Item added to cart") without interrupting the user or requiring a tap to dismiss it. React Native has no built-in cross-platform Toast component (Android has a native one, but iOS has none), so most apps use a library like react-native-toast-message that renders a consistent, custom-styled version across both platforms and auto-dismisses after a short timer.

JSX · Showing a Toast 
import Toast from 'react-native-toast-message';

Toast.show({
  type: 'success',
  text1: 'Added to cart!',
});

// Once near the app root: <Toast />

What is Lottie and how do you play a Lottie animation in React Native?

📖 Answer

Lottie is a format for vector animations exported from Adobe After Effects (typically by a designer) as a small JSON file, which lottie-react-native can then play back natively at full quality and smooth frame rates — without anyone writing a single line of animation code. It's how apps get those polished, complex onboarding illustrations or success checkmarks that would be impractical to hand-build with the Animated API, since a designer can iterate on the animation entirely outside the codebase.

JSX · Playing a Lottie File 
import LottieView from 'lottie-react-native';

<LottieView
  source={require('./success-check.json')}
  autoPlay
  loop={false}
  style={{ width: 150, height: 150 }}
/>

What is react-native-svg, and why can't you just use a normal <img> tag for SVGs?

📖 Answer

React Native has no concept of an <img> tag or a browser's SVG rendering engine at all — the platform's Image component can only display raster formats like PNG or JPEG. react-native-svg provides actual native SVG-drawing components (<Svg>, <Path>, <Circle>) that render vector graphics directly using native drawing APIs — letting icons and illustrations stay crisp at any size and remain easy to recolor programmatically, unlike a fixed-resolution bitmap.

JSX · Drawing an SVG Shape 
import { Svg, Circle } from 'react-native-svg';

<Svg height="100" width="100">
  <Circle cx="50" cy="50" r="40" fill="#0f766e" />
</Svg>

What is expo-splash-screen, and how do you control when it hides?

📖 Answer

The splash screen is the branded image shown the instant an app launches, while JavaScript is still loading and initializing. By default it auto-hides as soon as the app is ready to render, but expo-splash-screen lets you explicitly preventAutoHideAsync() and only call hideAsync() once your own setup (like fetching a user's auth state or loading fonts) has genuinely finished — avoiding an awkward flash of unstyled or empty content sneaking through before your app is truly ready to be seen.

JS · Manual Splash Control 
import * as SplashScreen from 'expo-splash-screen';

SplashScreen.preventAutoHideAsync();

async function prepareApp() {
  await loadFontsAndAuth();
  await SplashScreen.hideAsync(); // only now, once truly ready
}

What is expo-status-bar, and how do you customize the status bar?

📖 Answer

The status bar is the strip at the very top of the screen showing the time, battery, and signal icons — and its text color needs to contrast correctly against whatever your app's top content looks like. expo-status-bar lets you control its style (light text for dark backgrounds, dark text for light backgrounds), whether it's hidden entirely, and its background color (Android) — usually set once near the root of your app, or changed per-screen if different screens have very different background colors.

JSX · Setting Status Bar Style 
import { StatusBar } from 'expo-status-bar';

<StatusBar style="light" backgroundColor="#0f766e" />
// light text on a dark/colored top bar — readable, intentional

How do you load custom fonts with expo-font?

📖 Answer

Custom font files (.ttf/.otf) must be explicitly loaded before they can be referenced by name in a fontFamily style — unlike the web, where a font simply downloads and applies whenever it's ready. The useFonts hook from expo-font takes a map of font names to local file paths, and returns a boolean once loading is complete; you typically gate your entire app's render behind that boolean (often paired with the splash screen staying visible) so no text ever flashes briefly in the wrong default font.

JSX · useFonts Hook 
import { useFonts } from 'expo-font';

const [loaded] = useFonts({
  'Poppins-Bold': require('./assets/fonts/Poppins-Bold.ttf'),
});

if (!loaded) return null; // wait until fonts are ready
<Text style={{ fontFamily: 'Poppins-Bold' }}>Styled Text</Text>

What is expo-screen-orientation used for?

📖 Answer

expo-screen-orientation lets you programmatically lock an app (or a single screen) to a specific orientation — portrait-only, landscape-only, or allow both and respond to changes between them. This matters for things like a video player you want to allow rotating to landscape for full-screen viewing while keeping the rest of the app locked to portrait, or a game that's specifically designed for landscape play only.

JS · Locking Orientation 
import * as ScreenOrientation from 'expo-screen-orientation';

await ScreenOrientation.lockAsync(ScreenOrientation.OrientationLock.LANDSCAPE);
// later, when leaving the video screen:
await ScreenOrientation.lockAsync(ScreenOrientation.OrientationLock.PORTRAIT_UP);
Error Tracking & Code Quality

What is an Error Boundary in React, and how does it work in React Native?

📖 Answer

An Error Boundary is a special class component that catches JavaScript errors thrown anywhere in its child component tree during rendering, preventing the entire app from crashing to a blank white screen — instead, it can render a friendly fallback UI ("Something went wrong, please restart"). It works identically in React Native as it does on the web, using the same static getDerivedStateFromError / componentDidCatch lifecycle methods, since this is a React-level (not platform-level) mechanism.

JSX · Basic Error Boundary 
class ErrorBoundary extends React.Component {
  state = { hasError: false };
  static getDerivedStateFromError() { return { hasError: true }; }
  render() {
    if (this.state.hasError) return <Text>Something went wrong.</Text>;
    return this.props.children;
  }
}

What is Sentry, and why would you add crash/error reporting to a React Native app?

📖 Answer

Sentry is a third-party error-tracking service that automatically captures crashes and unhandled exceptions happening on real users' devices in production — including the full stack trace, device info, and the steps leading up to it — and sends them to a dashboard your team can review. Without it, you'd be entirely blind to bugs that only surface on specific devices or OS versions you don't personally own; with it, you often learn about and can fix a crash before users even bother reporting it themselves.

JS · Sentry Initialization 
import * as Sentry from '@sentry/react-native';

Sentry.init({ dsn: 'https://your-dsn@sentry.io/project' });
// Uncaught errors anywhere in the app now get reported automatically

What is Firebase Crashlytics, and how does it differ from Sentry?

📖 Answer

Crashlytics is Google's free crash-reporting tool, tightly integrated with the rest of Firebase (the same project you'd already use for FCM push notifications). It excels specifically at catching native-level crashes (a hard app crash, not just a caught JS exception) and is often the simpler default choice if your app already uses Firebase for other services. Sentry tends to offer richer JavaScript-specific tooling (source map support, release tracking, performance monitoring) and works equally well whether or not you use Firebase at all — the choice often comes down to which ecosystem the rest of your stack already lives in.

Conceptual · Sentry vs Crashlytics 
Sentry        → richer JS tooling, release/source map support, vendor-neutral
Crashlytics   → free, bundled with Firebase, strong native-crash detection
// Many teams use both — they catch slightly different failure types

Why is TypeScript commonly used in React Native projects, and what does it actually catch?

📖 Answer

TypeScript adds static type checking on top of JavaScript — you describe the expected "shape" of your data (a function's parameters, a component's props, an API response), and your editor/build process flags mismatches before the app ever runs, rather than discovering them as a runtime crash on a user's device. Common things it catches: passing the wrong type of prop to a component, typos in object keys, forgetting to handle a possible null/undefined value, or calling a function with the wrong number of arguments. It's now the default for new Expo/React Native projects rather than an optional add-on.

TS · Catching a Bug Before Runtime 
type UserCardProps = { name: string; age: number };

function UserCard({ name, age }: UserCardProps) {
  return <Text>{name} is {age}</Text>;
}

<UserCard name="Sam" age="twenty" />
// ❌ TypeScript error caught immediately: age expects a number, not a string

What do ESLint and Prettier do, and why use both together?

📖 Answer

ESLint analyzes your code for potential bugs and bad practices — an unused variable, a missing dependency in a useEffect, a possible infinite loop pattern — flagging issues that could cause real problems. Prettier is purely about formatting: consistent indentation, quote style, line length — it has no opinion on whether your logic is correct, only on how it looks. Running both together means ESLint focuses entirely on catching real bugs while Prettier handles all formatting bickering automatically, so teams stop arguing over tabs-vs-spaces in code review.

Conceptual · Different Jobs 
ESLint:    "This variable is never used" — catches LOGIC issues
Prettier:  "This line should use double quotes, not single" — FORMATTING only
// Together: clean code AND consistent style, with no manual nagging
210

What are common causes of memory leaks in React Native apps?

📖 Answer

The most frequent culprit is forgetting to clean up a subscription, listener, or timer when a component unmounts — a setInterval, a location watch, an event listener, or a still-pending network request that calls setState after the component is already gone, which can also throw a "cannot update an unmounted component" warning. The fix is almost always the same pattern: return a cleanup function from useEffect that cancels/removes whatever was set up, ensuring nothing keeps running or holding memory once a screen is no longer in use.

JS · The Cleanup Pattern 
useEffect(() => {
  const interval = setInterval(() => console.log('tick'), 1000);
  const sub = someAPI.subscribe(handleEvent);

  return () => {            // ✅ ALWAYS clean up
    clearInterval(interval);
    sub.unsubscribe();
  };
}, []);

What are some techniques for reducing React Native app bundle size?

📖 Answer

Common techniques include: enabling Hermes (its bytecode format is more compact than raw JS), removing unused dependencies and dead code, compressing image assets and using appropriately-sized images rather than oversized originals, lazy-loading screens/features that aren't needed immediately rather than bundling everything upfront, and avoiding accidentally importing an entire library when you only need one small function from it (checking for tree-shaking support). A smaller bundle means faster downloads, faster installs, and often faster app startup too.

Conceptual · Bundle Size Checklist 
✅ Hermes enabled (smaller bytecode vs raw JS parsing)
✅ Remove unused npm packages and dead code
✅ Compress/resize image assets appropriately
✅ Lazy-load screens not needed on first launch
✅ Import only what you need: { debounce } from 'lodash-es', not all of lodash

What is OAuth-based social login, and how does expo-auth-session help implement it?

📖 Answer

OAuth social login lets users sign in using an existing account (Google, GitHub, etc.) without ever typing a new password into your app — your app redirects the user to the provider's own login page, the user authenticates there, and the provider redirects back with proof of identity (a token) your app can trust. expo-auth-session handles the tricky parts of this flow on mobile — opening the provider's login page in a secure in-app browser, capturing the redirect back into your app, and exchanging the authorization code for a usable token.

JS · Basic Auth Session Flow 
import * as AuthSession from 'expo-auth-session';

const result = await AuthSession.startAsync({
  authUrl: 'https://provider.com/oauth/authorize?...',
});
// result contains the auth code/token once the user signs in & redirects back

What is "Sign in with Apple," and when is it required?

📖 Answer

Apple's own privacy-focused sign-in option (similar in spirit to "Sign in with Google") lets users authenticate using their Apple ID, optionally hiding their real email behind a private relay address. Apple's App Store guidelines require offering Sign in with Apple as an option if your app offers any other third-party social login (like Google or Facebook) — you can't offer Google sign-in alone without also providing Apple's equivalent, making this one of the few login methods that isn't purely optional for many apps targeting iOS.

JS · Sign in with Apple 
import * as AppleAuthentication from 'expo-apple-authentication';

const credential = await AppleAuthentication.signInAsync({
  requestedScopes: [
    AppleAuthentication.AppleAuthenticationScope.FULL_NAME,
    AppleAuthentication.AppleAuthenticationScope.EMAIL,
  ],
});

What are in-app purchases, and what does a library like RevenueCat provide?

📖 Answer

In-app purchases (IAP) let users buy digital goods or subscriptions directly inside your app, processed through Apple's or Google's own payment systems (required by store policy for digital goods — you generally can't route around them with your own payment processor for this category). Handling raw IAP APIs directly is notoriously fiddly across both platforms; RevenueCat is a popular service that wraps both platforms' purchase APIs into one consistent interface, handles receipt validation on its servers, and gives you a dashboard for subscription analytics — saving teams from re-implementing a lot of error-prone payment plumbing themselves.

JS · RevenueCat Purchase Flow 
import Purchases from 'react-native-purchases';

const offerings = await Purchases.getOfferings();
const purchase = await Purchases.purchasePackage(offerings.current.monthly);
// RevenueCat handles receipt validation + cross-platform entitlement tracking

What is the Linking API used for besides deep linking into your own app?

📖 Answer

React Native's built-in Linking module also lets your app open other apps and system features by URL — opening a web page in the device's browser, launching the phone dialer with a number pre-filled, opening the Mail app with a draft email, or opening another installed app via its own custom URL scheme. It's the same general mechanism deep linking uses, just pointed outward instead of inward — your app handing off to somewhere else, rather than receiving a handoff.

What is a background task (expo-background-fetch / expo-task-manager), and what are its limitations?

📖 Answer

Background tasks let your app run a small piece of JavaScript periodically even when it's not open — useful for syncing data or checking for updates without requiring the user to manually reopen the app. expo-task-manager registers the task, and expo-background-fetch schedules it to run roughly every so often. The key limitation: both iOS and Android tightly control background execution to protect battery life — your task isn't guaranteed to run at an exact interval, may be skipped entirely under low-battery or restrictive conditions, and typically has only a short window (a few seconds to a couple minutes) to do its work before being forcibly suspended.

JS · Registering a Background Task 
import * as TaskManager from 'expo-task-manager';
import * as BackgroundFetch from 'expo-background-fetch';

TaskManager.defineTask('SYNC_TASK', async () => {
  await syncLatestData();
  return BackgroundFetch.BackgroundFetchResult.NewData;
});

await BackgroundFetch.registerTaskAsync('SYNC_TASK', { minimumInterval: 900 });
// "Every ~15 min" is a request, not a guarantee — the OS decides the real timing

What is a custom hook, and why would you extract one?

📖 Answer

A custom hook is simply a regular JavaScript function (whose name starts with use, by convention) that calls other React hooks internally — letting you package up a reusable piece of stateful logic (like "fetch and track loading state for X" or "subscribe to network connectivity") once, and reuse it across multiple components without copy-pasting the same useState/useEffect combination everywhere. It's the modern replacement for the older Higher-Order Component pattern, and is generally considered easier to read and compose.

JS · A Reusable Custom Hook 
function useNetworkStatus() {
  const [isConnected, setIsConnected] = useState(true);
  useEffect(() => {
    const sub = NetInfo.addEventListener((state) => setIsConnected(state.isConnected));
    return () => sub();
  }, []);
  return isConnected;
}

// Reused anywhere: const isConnected = useNetworkStatus();

What is the App Store Connect / Google Play Console review process like for a new submission?

📖 Answer

After building your app (typically via eas build) and uploading it, you fill out store-listing metadata (screenshots, description, privacy policy URL, content rating) inside App Store Connect (Apple) or Google Play Console (Android), then submit for review. Apple's review is manual and can take anywhere from a few hours to a few days, scrutinizing functionality, permission justifications, and guideline compliance fairly strictly — rejections for vague reasons aren't uncommon. Google's review is largely automated and generally faster, though policy violations (especially around sensitive permissions like background location) still trigger manual scrutiny on both platforms.

Bash · Build & Submit via EAS 
eas build --platform ios --profile production
eas submit --platform ios
// Similarly for Android with --platform android

What is "runtimeVersion" and update channels in EAS, and why do they matter for OTA updates?

📖 Answer

Since OTA (EAS) updates can only ship JS-only changes, EAS needs a way to guarantee a given JS bundle is actually compatible with the native code already installed on a user's device — that's what runtimeVersion tracks: devices only receive OTA updates that match their installed native runtime, preventing a mismatch crash. Channels (like "production," "preview," "staging") let you target different groups of builds with different updates — so you can push a fix to internal testers on a "preview" channel without it touching real production users at all, until you're confident and promote it.

Bash · Channel-Targeted Update 
eas update --branch preview --message "Testing new checkout flow"
// Only devices built with channel "preview" pointing to this branch receive it
// runtimeVersion ensures it only applies to a compatible native build

What are adaptive icons, and how do app icon/splash configurations differ across platforms?

📖 Answer

iOS expects a single, fixed-shape square icon image that the OS automatically masks into a rounded square. Android, however, uses adaptive icons — you supply a separate foreground layer (your logo/symbol) and a background layer, and the OS dynamically applies different mask shapes (circle, squircle, rounded square) depending on the device manufacturer's launcher theme, so your icon looks correct everywhere instead of being clipped oddly. In a Managed Expo project, both are configured declaratively inside app.json rather than manually placed into native image asset folders.

JSON · Icon Config in app.json 
{
  "expo": {
    "icon": "./assets/icon.png",
    "android": {
      "adaptiveIcon": {
        "foregroundImage": "./assets/adaptive-icon.png",
        "backgroundColor": "#0f766e"
      }
    }
  }
}
